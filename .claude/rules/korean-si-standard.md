## 화면설계서 필수 내용 (SCR-NNN.md)

### 1. 화면 목적
- 이 화면이 왜 존재하는가
- 사용자가 이 화면에서 달성하려는 것

### 2. 와이어프레임
- ASCII 또는 Mermaid로 레이아웃
- 영역별 역할 표시

### 3. 상세 기능 목록
- 화면 내 모든 요소 (입력, 버튼, 테이블, 필터 등)
- 각 요소의 데이터 타입, 기본값, 필수 여부

### 4. 상호작용 (Interaction)
- 클릭, 호버, 드래그, 키보드 등
- 각 상호작용의 트리거 → 결과

### 5. 화면 내 프로세스
- 버튼 클릭 시 조건 분기 (예: "저장 버튼: 유효성 통과 → API 호출, 실패 → 에러 표시")
- 상태 전이 (로딩 → 성공/실패 → 결과 표시)
- 조건부 표시/숨김 로직

### 6. API 연동
- 호출 시점, 엔드포인트, 요청/응답 구조
- 에러 처리 (어떤 에러 → 어떤 화면 반응)

### 7. 엣지 케이스
- 빈 데이터, 권한 없음, 네트워크 에러, 대량 데이터

## IA (Information Architecture) 필수 내용

### 도메인 정리
- 각 도메인이 **왜** 존재하는가 (목적)
- 도메인 간 관계와 경계

### 메뉴 구조
- 각 메뉴가 **무엇을 위해** 있는가
- 메뉴 접근 권한 조건

### 프로세스 제약조건
- 요구사항을 구현할 때 지켜야 할 비즈니스 제약
- 순서 의존성 (A가 완료되어야 B 가능)
- 데이터 정합성 규칙
- 동시성/권한 제약
