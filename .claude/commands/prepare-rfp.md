docs/00_srs/에 있는 원본 PDF와 복붙 .md를 교차 검증하여 정제된 RFP를 완성한다.
design-orchestrator 실행 전에 반드시 먼저 실행한다.

---

## 실행 전 필수 확인
1. docs/00_srs/ 에 원본 PDF 존재 확인
2. docs/00_srs/ 에 사용자가 복붙한 .md 파일 존재 확인
3. 둘 다 없으면 사용자에게 요청

---

## Step 1: 복붙 .md 구조화
- 사용자가 복붙한 .md 파일 읽기
- 섹션별로 분리 (목차, 개요, 요구사항, 표, 별첨 등)
- 깨진 표 복원 (컬럼 정렬, 헤더 복구)
- 번호체계 정리 (1.1, 1.2... 계층 복원)

## Step 2: PDF 읽기 (🔀 병렬 — Step 1과 동시)
- 서브에이전트가 원본 PDF를 /pdf 스킬로 읽기
- 전체 구조/순서/섹션 목차 추출
- 페이지별 내용 요약

## Step 3: 교차 검증
- Step 1 결과 (구조화된 복붙) ↔ Step 2 결과 (PDF 읽기) 비교
- 검증 항목:
  | 항목 | 체크 |
  |------|------|
  | 섹션 수 일치 | 복붙에 있는 섹션이 PDF에도 있는가 |
  | 표 데이터 일치 | 복붙 표의 행/열 수가 PDF와 맞는가 |
  | 누락 섹션 | PDF에 있는데 복붙에 없는 부분 |
  | 순서 일치 | 내용 순서가 원본과 동일한가 |
- 불일치 발견 시 목록화

## Step 4: 사용자 확인
- 불일치 목록을 사용자에게 보여주기:
  ```
  ⚠️ 교차 검증 결과:
  - 섹션 3.2 표: PDF에는 12행인데 복붙에는 10행 (2행 누락 가능)
  - 섹션 5: PDF에 있는데 복붙에 없음
  - 별첨 A: 복붙에만 있음 (PDF에서 못 찾음)

  확인해주세요: (수정 / 무시 / 원본 재복붙)
  ```

## Step 5: 정제본 생성
- 검증 완료된 내용을 최종 정리
- docs/00_srs/rfp-final.md 생성
- 파일 구조:
  ```markdown
  # RFP 정제본
  > 원본: {원본 PDF 파일명}
  > 교차검증: 완료 ({일시})
  > 불일치: {N}건 확인됨 → 해결됨

  ## 1. {섹션명}
  ...
  ```

## Step 6: 외부 시스템 연동 사전 탐지
- rfp-final.md에서 외부 시스템/인터페이스 관련 내용 스캔
- 탐지 키워드: SAP, ERP, CRM, API, 연동, 연계, 인터페이스, 전송, 수신, 배치, 외부, 타시스템
- **발견 시:**
  - docs/03_data/interface-spec.md 스텁 생성
  - RFP에서 파악 가능한 만큼 채움:
    ```markdown
    # 외부 시스템 인터페이스 (사전 탐지)
    > 출처: rfp-final.md 분석
    > 상태: 스텁 (Phase 5에서 상세화 예정)

    | IF-ID | 대상 시스템 | 연동 유형 | 방향 | RFP 근거 | 상세 여부 |
    |-------|-----------|----------|------|---------|----------|
    | IF-SAP-001 | SAP (추정) | RFC/API | 양방향 | "SAP 재무모듈 연동" (p.12) | ❌ 미상세 |
    | IF-PG-001 | 결제 (추정) | 외부 API | 송신 | "온라인 결제 기능" (p.8) | ❌ 미상세 |
    ```
  - 사용자에게 확인: "이런 외부 연동이 있는 것 같습니다. 맞나요? 추가할 것 있나요?"
- **미발견 시:**
  - 사용자에게 질문: "외부 시스템 연동이 있나요? (SAP, 결제, 알림, 타 시스템 등)"
  - 없으면 넘어감 (interface-spec.md 미생성)
